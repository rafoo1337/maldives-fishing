# 16 — Платежи и юридические аспекты

> **Дисклеймер:** бизнес-ресёрч для планирования (июнь 2026), **НЕ юридическая консультация**. Перед запуском верифицировать у юристов в РФ, ОАЭ, на Мальдивах. Часть данных по 2026 — законопроекты/анонсы регуляторов (помечено).
> Контекст: основная компания — **Дубай (ОАЭ)**, есть компания на **Мальдивах**; много клиентов из **РФ** + ЕС/Залив/Азия. Сайт — **лид→менеджер, без онлайн-оплаты**.

## Главный вывод
Модель «лид→менеджер без онлайн-оплаты» **сама снимает большую часть платёжного и крипто-рекламного риска**. Дальше — как не создать риск там, где его легко избежать.

## 1. РФ 152-ФЗ и сбор данных на сайте
- **Подпадает ли иностранная компания:** да, с высокой вероятностью. С 01.09.2022 закон экстерриториален: если деятельность «направлена на РФ» и собираются ПДн россиян (имя/тел/email = ПДн). **Признаки направленности:** русский язык сайта, домен `.ru/.рф`, цены/оплата в рублях, поддержка на русском, реклама на РФ.
- **Локализация (с 01.07.2025 ужесточена):** первичный сбор/хранение ПДн россиян — в БД **на территории РФ** (трансграничная передача потом возможна).
- **Штрафы (с 30.05.2025):** неуведомление РКН 100–300 тыс ₽; **нарушение локализации 1–6 млн ₽** (повтор до ~18 млн); обработка без согласия 300–700 тыс; утечки до 15 млн; **повторная утечка — оборотный 1–3% выручки (до 500 млн ₽)**. Главный рычаг против иностранцев без активов в РФ — **блокировка сайта по IP/домену** (прецедент LinkedIn).
- **Митигирование (от простого):**
  1. **Не собирать ПДн под РФ-аудиторию формой.** Для РФ-гео — кнопка «написать менеджеру в WhatsApp/Telegram» (переписку инициирует пользователь, ПДн в БД сайта не пишутся → формально нет «первичного сбора»). Серая зона, но риск кардинально ниже.
  2. **Гео-разделение форм:** РФ → мессенджер без полей; форма с полями → только EN/не-РФ.
  3. Полный комплаенс (российский хостинг первичной БД + согласие + уведомление РКН) — дорого, для лид-формы обычно избыточно.

## 2. Приём оплат от РФ-клиентов (ОАЭ/Мальдивы)
- **«Мир»** — не принимается. **UnionPay** РФ-банков — нестабильно после санкций 11.2024, в бизнес-процесс не закладывать.
- **Visa/MC не-РФ банков** (СНГ/Турция/ОАЭ-резидентство) — работают; но это карта самого клиента, не наша схема.
- **SWIFT** из подсанкционных РФ-банков заблокирован; **СПФС** под предупреждением OFAC — рискованно для компании, обслуживающей ещё и ЕС.
- **Крипта (USDT)** — на практике самый рабочий трансграничный канал, НО см. ограничения по рекламе/ОАЭ ниже.
- **Посредники/турагенты** в РФ/СНГ (собирают рубли локально) — снимают трение, но комиссия + due-diligence.
- **⚠️ Комплаенс-риск:** выстраивание рельсов «специально под РФ» (особенно через подсанкционные банки/СПФС) может ударить по долларовым корсчетам банка в ОАЭ (de-risking). **Безопаснее:** оплату инициирует клиент со своей стороны, компания выставляет инвойс в **USD**. Проверить с комплаенс-офицером банка ОАЭ.

## 3. Крипта — где можно заявлять «принимаем крипту»
**Легальность приёма ≠ право рекламировать.**

| Юрисдикция | Принимать | Открыто рекламировать |
|---|---|---|
| **ОАЭ (VARA)** | Да, через **VARA-лицензированного провайдера**; не-дирхамовые стейблы (USDT) — только через одобренных провайдеров (с 07.2025) | Да, в рамках лицензии |
| **Мальдивы** | Серая зона; MMA не выдавала разрешений, предупреждала против | **Не рекомендуется** афишировать |
| **ЕС (MiCA)** | Через лицензию; переходный период до 01.07.2026 | Условно, регулируется |
| **РФ** | Внутренние крипто-платежи **запрещены** (трансграничные — легализуют как имущество) | **ЗАПРЕЩЕНО** — готовится прямой запрет рекламы крипты/упоминания монет |

**Вывод:** плашку «We accept crypto» — максимум на **EN-версии для не-РФ**, только через VARA-схему; на **RU-версии — не рекламировать крипту вообще**. Приём USDT от конкретного клиента — приватно через менеджера + проверка с юристом ОАЭ.

## 4. Платёжные шлюзы по регионам (на будущую фазу с депозитами)
| Регион | Шлюзы | Заметки |
|---|---|---|
| **ОАЭ/Залив** (основной эквайринг) | Network International (N-Genius), Telr, PayTabs, Checkout.com, Stripe (UAE), Tap | N-Genius силён для hospitality; Stripe — для подписок/антифрод; Telr/PayTabs — быстрый онбординг, Mada/BNPL |
| **Глобально** | Stripe, Checkout.com, Adyen, PayPal | Visa/MC/Amex, Apple/Google Pay (Apple Pay +15–30% конверсии мобайла) |
| **РФ** | ЮKassa, Т-Касса, CloudPayments | **Только с РФ-юрлицом/счётом** → тянет 152-ФЗ + налоги. Обычно не оправдано |
| **Крипто (ОАЭ)** | VARA-провайдеры (напр. Pallapay) с авто-конвертацией | Только лицензированный провайдер + лицензия покрывает VA |

## 5. Рекомендация для сайта / воронки
**Показывать:** контент + цены **ориентировочно в USD** (не рубли — маркер «направленности на РФ»); CTA «связаться с менеджером»; **для РФ-гео — кнопка в мессенджер без полей**; EN-версия может иметь обычную форму.
**НЕ показывать:** «принимаем USDT/крипту» на RU-версии; цены в рублях; `.ru` как основной домен; русскоязычную форму с телефоном без комплаенса; онлайн-оплату, которой нет.
**Отдать менеджеру (приватно):** подбор способа оплаты под клиента; инвойс в USD от компании ОАЭ; деньги инициирует клиент; KYC посредников.
**До запуска провалидировать:** (1) юрист 152-ФЗ — гео-логика форм; (2) комплаенс банка ОАЭ — каналы от РФ-клиентов; (3) юрист ОАЭ по VA — крипта/провайдер.
