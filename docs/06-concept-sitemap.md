# 06 — Концепция и сайтмап

> Обзорная концепция. **Канонический детальный сайтмап и слаги — в [12 — Информационная архитектура](12-information-architecture.md)** (при расхождении слагов верно 12). Принцип: взять рабочую модель эталонов и закрыть их дыры (мультиязык, дефицит-визуализация, видео, золотая середина «цена + лид»).

## Принципы
- **Fishing-first:** рыбалка — герой, отель — крючок.
- **EN-first + мультиязык** (EN → RU → AR/DE/FR/ZH по приоритету рынков) с корректным `hreflang`.
- **Лид → менеджер.** Без онлайн-оплаты и личного кабинета (решение клиента).
- **Дефицит как фича:** сезон/календарь, «осталось N окон», предоплата 50%.
- **Доверие через контент:** видео-трофеи, датированные catch-reports, капитаны-личности, отзывы с именами.

## Воронка (главная логика)
```
HERO «На Мальдивах есть рыбалка мирового уровня» (90% не знают)
   ↓ желание: видео трофеев (GT, марлин, тунец), эмоция
   ↓ 3 вида рыбалки: Fly · Big game · Sport (выбор «своего»)
   ↓ Атоллы & Отели: где, какой штиль/логистика, какой отель
   ↓ Пакеты «от $»: матрица жильё × формат рыбалки (+ «Fisher + Partner»)
   ↓ Дефицит/сезон: мало катеров, календарь, предоплата 50%
   ↓ ЛИД (многошаговый): направление → даты → Anglers + Leisure → контакты
   ↓ Менеджер: WhatsApp + SLA «ответим за час», индивидуальный расчёт
```

## Карта сайта
- **/** — Home (hero, оффер, козыри локации, превью видов/пакетов/контента, CTA Enquiry).
- **/fishing** — Виды рыбалки (обзор) → **/fishing/fly**, **/fishing/big-game**, **/fishing/sport** (для кого, что ловим, сезон, для новичка/профи).
- **/destinations** — Атоллы & Отели (Хувадху и др.; штиль/логистика/выбор отелей; страница на отель/атолл).
- **/packages** — Пакеты «от $» (матрица жильё × формат; «Fisher + Partner» для пар; «Стань автором своего тура» — кастом).
- **/why-maldives** — Почему Мальдивы > Сейшел (закрытые атоллы/штиль, логистика, семья рядом, рекорд Гиннеса).
- **/content** — Контент-хаб: **catch-reports** (датированные), видео, блог, гайд (когда ехать/что выбрать).
- **/safety** — Безопасность на борту + обучалки (видео-правила, основы рыбалки).
- **/about** — Команда, капитаны (личности, опыт 17 лет, 40+ отелей), conservation.
- **/faq** — Вопрос-ответ (укачивание, снасти, новичок, что взять).
- **/enquiry** — Многошаговая заявка → менеджер.
- **/[lang]/…** — мультиязычные версии (EN дефолт).
- Контакты: 2 номера (MV + RU) + WhatsApp, на разных языках — разные номера.

## Лид-форма (по образцу Blue Safari, усиленная)
1. Интересующее направление / вид рыбалки (мультивыбор)
2. Год → месяц / сезон
3. **Anglers (1–10) + Leisure Guests (0–5)** ← монетизация пар/семей
4. Контакты → переход к менеджеру (WhatsApp-deeplink с предзаполненным сообщением) + обещание SLA «ответим за час»

## Конфигуратор-как-презентация (опц., Фаза 1 → C)
Лёгкий выбор: вид рыбалки → отель/атолл → «от $X за 7н/6д» → лид. Не букинг, а **презентация-вилка**, которая тянет в диалог с менеджером. Решить в Discovery (см. [07](07-decisions.md)).

## Фичи доверия (закрывают дыры конкурентов)
- Датированные **catch-reports** с видами/размерами/именами гостей (как Alphonse/Boombastic, но **с видео**).
- Профили **капитанов** как личностей + «мы здесь живём 3.5 года».
- Отзывы с именами/странами; conservation-нарратив как премиум-рычаг.
- Визуализированный **дефицит**: календарь сезона, «осталось N окон», предоплата/штрафы.

## Технический стек — предложение
- **Frontend:** **Next.js (App Router) + TypeScript + Tailwind**, i18n-роутинг (`next-intl`), `hreflang`, image/video-оптимизация. Премиальный визуал, быстрый мобайл (дыра Boombastic — тяжёлые страницы).
- **Контент:** headless-CMS (catch-reports/блог редактирует клиент/Влад без разработчика).
- **Лиды:** форма → CRM + WhatsApp; email-уведомления менеджеру.
- **Деплой:** Vercel (превью-ссылки на каждую правку — удобно показывать клиенту с телефона).
- **Почему не WordPress / почему задел под моб. приложение:** общий бэкенд/данные → масштабирование в приложение (Модель 2) без переписывания; WP упрётся в кастомную логику и нагрузку. (Аргумент Адама на созвоне — фиксируем в КП.)

> Следующий шаг после согласования концепции — **живой демо-сайт** (Next.js) на этом стеке, задеплоить на Vercel → ссылка для показа клиенту прямо с телефона.
