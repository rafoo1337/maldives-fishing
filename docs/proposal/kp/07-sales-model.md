# Глава 7 — Как сайт продаёт (модель)

<!-- 1 страница. Модель конверсии: лид → менеджер, не букинг. -->

## Заголовок
**Сайт собирает «тёплый» лид — закрывает менеджер**

Для вашего чека и авторского продукта это единственно верная модель: без обезличенного онлайн-магазина и без личного кабинета. Сайт делает тяжёлую работу (вызвать желание, закрыть возражения, отфильтровать), а сделку доводит живой человек.

## Воронка
```
Hero «здесь есть рыбалка мирового уровня» (90% не знают)
 → желание: видео трофеев, эмоция
 → 3 вида рыбалки (выбор «своего»)
 → атоллы и отели
 → пакеты «от $» (матрица + лёгкий конфигуратор)
 → дефицит/сезон
 → заявка (направление → даты → рыбаки+сопровождающие → контакты)
 → менеджер (WhatsApp, «ответим за час»)
```

## Почему НЕ «как booking»
Живой выбор дат и наличия — это лицензия партнёра (~$10 000), синхронизация, юр-сложности в РФ и тяжёлая поддержка. Для вашей модели не нужно (это «вторая модель» на будущее).

## Что делаем вместо — «конфигуратор-презентация»
Гость выбирает: **вид рыбалки → отель/атолл → формат** и получает **вилку «от $X»**, после чего уходит в заявку — уже с понятным контекстом для менеджера. **Ощущение booking — без его сложности и рисков.** (Лёгкая версия — в рекомендуемый тариф B.)

## «Тёплый» лид
Форма квалифицирует ещё до контактов: направление → год/месяц → число рыбаков + сопровождающих → контакты. Менеджер получает заявку с готовой картиной — и сразу говорит по делу.

## «Стань автором своего тура»
Для тех, кто не нашёл готовый пакет — отдельный путь кастом-заявки. Ваша же идея, оформленная в воронку.
