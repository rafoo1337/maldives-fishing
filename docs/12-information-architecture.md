# 12 — Информационная архитектура (постранично)

> Детальная версия [06 — Концепция и сайтмап](06-concept-sitemap.md). Структура согласована с SEO-pillar'ами ([14](14-seo-strategy.md)), позиционированием ([11](11-positioning-messaging.md)) и аудиторией ([10](10-audience-personas.md)). Это спека, по которой можно проектировать дизайн и собирать сайт.

## Принципы
- **Fishing-first:** рыбалка — герой, отель — крючок/апселл.
- **EN-first + мультиязык** (`/en/ /de/ /ru/`), корректный `hreflang`, быстрый мобайл.
- **Лид → менеджер**, без онлайн-оплаты/личного кабинета.
- **Дефицит и доверие — сквозные** переиспользуемые блоки.
- **Под AI-поиск (GEO):** вопросные заголовки, FAQ-блоки, schema.

## Карта сайта
```
/[lang]/
├── /                         Home
├── /fishing-in-maldives      PILLAR 1 — гайд (трафик-магнит) + споки (season, atolls, rules, cost…)
├── /big-game-fishing         PILLAR 2 — big game/troll/jig/popping (+ видовые: GT, tuna, marlin, wahoo)
├── /fly-fishing              PILLAR 3 — флай/флэтс (GT, триггерфиш, bonefish-бонус)
├── /packages                 PILLAR 4 ⭐ — пакеты «рыбалка + отель» (+ конфигуратор-презентация)
│   ├── /packages/liveaboard
│   ├── /packages/resort-stay
│   └── /packages/private-yacht
├── /for/couples              Сегмент — пары (Fisher + Partner)
├── /for/families             Сегмент — семьи (безопасно, дети)
├── /for/corporate            Сегмент — корпоратив/ретриты (высокий чек)
├── /atolls                   Атоллы & отели-партнёры (гео-страницы)
│   └── /atolls/{atoll}        (Хувадху и др.)
├── /journal                  Контент-хаб: catch-reports, видео, блог, гайды ([13](13-content-strategy.md))
├── /safety                   Безопасность на борту + обучалки (видео)
├── /about                    Бренд, капитаны-личности, conservation, founder story
├── /faq                      Вопрос-ответ (+ FAQPage schema)
└── /enquiry                  Многошаговая заявка → менеджер
```

## Глобальные элементы
- **Шапка:** лого · меню (Fishing ▸ [Big game/Fly/Sport], Packages, Atolls, Journal, About) · переключатель языка (ручной, без принудительного гео-редиректа) · **сквозной CTA «Plan your trip / Talk to a concierge»**.
- **Контакты:** 2 номера (MV + RU) + **WhatsApp-deeplink с предзаполненным сообщением**; для RU-гео — мессенджер вместо формы (см. [16](16-payments-legal.md)).
- **Футер:** виды рыбалки, пакеты, журнал, about, FAQ, контакты, соцсети (YouTube/IG), политика/условия.
- **Sticky mobile CTA** (звонок/WhatsApp/заявка).

## Постранично (ключевое)

### Home `/`
- **Hero:** видео-трофей (GT/тунец) + слоган «The Maldives You Were Never Sold» + CTA. Сразу закрывает «а есть ли тут рыбалка».
- Секции: 3 вида рыбалки (карточки) → почему Мальдивы (закрытые атоллы/штиль, логистика, семья рядом) → пакеты «от $» (превью Pillar 4) → дефицит/сезон → контент-превью (catch-reports/видео) → соцдоказательство (трофеи, отзывы) → капитаны → CTA-заявка.
- **GEO:** definition-блок «Is there fishing in the Maldives?» + ответ в первых 40–60 словах.

### Виды рыбалки (Pillar 2/3 + Sport)
Каждая: для кого (новичок→профи) · что ловим (виды из [фактуры](reference/facts-maldives-fishing.md)) · сезон/техника · фото/видео трофеев · «как проходит день» (сцена) · безопасность · CTA «собрать как пакет». **Spoke-страницы** под видовые/техничные ключи.

### Pillar 1 `/fishing-in-maldives` (гайд)
Answer-first структура под GEO/SEO: вопросные H2 (season, atolls, license/rules, cost, fly vs big game), таблица «сезон×вид×техника×атолл», FAQ-блок. В каждый раздел — мост на **Packages**.

### Packages `/packages` ⭐ (главная конверсионная)
- **Матрица пакетов** (по образцу Blue Safari, но прозрачнее): формат × проживание → **«from $X»** + что входит (гайд, снасти, трансферы, питание) + что отдельно. Топ-опции — «от $», не «on request».
- **Конфигуратор-как-презентация** (НЕ букинг): вид рыбалки → отель/атолл → длительность → **вилка «от $X»** → переход в заявку с уже заполненным контекстом. Без реального календаря наличия (см. [09 §2](09-inconsistencies.md)).
- Пакет **«Fisher + Partner»** (рыбачит один, второй — спа/дайвинг) — отдельный SKU под пары/семьи.
- «**Стань автором своего тура**» — кастом-путь.

### Сегментные `/for/couples`, `/for/families`, `/for/corporate`
Под драйверы из [10](10-audience-personas.md): пары («он на флэтах, вы на закате»), семьи (безопасность детей, night/reef fishing, первая рыба), корпоратив (приватность, тимбилдинг, серьёзная отдельная страница — высокий чек). Каждая → свой пакет + заявка.

### Atolls `/atolls/{atoll}`
Гео-страницы (Хувадху и др.): чем силён атолл, какие виды/точки, отели-партнёры, логистика. Бьёт гео-long-tail (`north male/ari atoll fishing`) и связывает «рыбалка ↔ отель».

### Journal `/journal` (контент-хаб)
Catch-reports (датированные, с видом/размером/именем), видео-библиотека (переиспользовать 40+ обзоров + Влад), блог/гайды (SEO), сезонные wrap-up. Двигатель доверия и органики. → [13](13-content-strategy.md)

### Safety `/safety` · About `/about` · FAQ `/faq`
Safety — видео-правила на борту + обучалки (доверие, видео>бумажки). About — капитаны-личности, 17 лет опыта, 40+ отелей, conservation, founder story. FAQ — вопрос-ответ + FAQPage schema (GEO).

### Enquiry `/enquiry` (многошаговая заявка)
1. Направление / вид рыбалки (мультивыбор)
2. Год → месяц / сезон
3. **Anglers (1–10) + Leisure Guests (0–5)** ← монетизация пар/семей
4. Контакты → менеджер (WhatsApp-deeplink, **SLA «ответим за час»**)
- **RU-гео:** шаг с полями заменяется кнопкой в мессенджер (см. [16](16-payments-legal.md)).

## Воронка
```
HERO (90% не знают) → желание (видео/трофеи) → 3 вида рыбалки → почему Мальдивы
→ Packages «от $» (матрица + конфигуратор) → дефицит/сезон → многошаговый лид → менеджер (WhatsApp, SLA)
```

## Переиспользуемые блоки (компоненты)
- **Scarcity-блок:** «6 отелей · мало катеров · ноябрь/март уже проданы · предоплата 50%» + календарь сезона.
- **Trust-блок:** трофеи с метаданными · капитаны · отзывы с именами · conservation.
- **Catch-report-карточка** (вид/размер/дата/гид/фото).
- **CTA-блок** «Talk to a fishing concierge» (везде).

## Технические требования к ИА
- SSR/SSG (под SEO/GEO), schema на типах страниц, `hreflang`+`x-default`, sitemap по языкам, скорость на мобайле, доступность. Стек — [15](15-tech-stack.md).
